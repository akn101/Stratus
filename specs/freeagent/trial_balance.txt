          <h1>Trial Balance</h1>
<p><em>Minimum access level</em>: <code>Tax, Accounting &amp; Users</code>, unless stated otherwise.</p>
<h2 id="get-the-trial-balance-summary">Get the trial balance summary</h2><pre class=prettyprint><code class=>GET https://api.freeagent.com/v2/accounting/trial_balance/summary
</code></pre>
<p>Date Filters</p>
<pre class=prettyprint><code class=>GET https://api.freeagent.com/v2/accounting/trial_balance/summary?to_date=2014-05-01
GET https://api.freeagent.com/v2/accounting/trial_balance/summary?from_date=2014-01-01&amp;to_date=2014-05-01
</code></pre>
<ul>
<li><code>from_date</code> - Date in YYYY-MM-DD format (optional)</li>
<li><code>to_date</code> - Date in YYYY-MM-DD format</li>
</ul>
<h3>Response</h3>
<p>If no dates are specified, the summary is for the current date.</p>

<p>If only <code>to_date</code> is specified, the summary is for a period containing the
specified date. The trial balance figures are from the beginning of the
accounting period up to the specified date.</p>

<p>If both <code>from_date</code> and <code>to_date</code> are specified, the summary is for a period
with the specified custom range.</p>
<h4>Note regarding response attributes</h4>
<p>The <code>display_nominal_code</code> attribute always returns the full category code as it&#39;s displayed on accounting reports
in the FreeAgent web UI, and should be used in most circumstances instead of <code>nominal_code</code>. The <code>nominal_code</code>
attribute maintains legacy behaviour where, for bank account and user categories (codes 750 and 900-910), the second
part of the code refers to the ID of the related resource (bank account or user) rather than the category sub-code.</p>
